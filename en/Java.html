Student Records

The program is designed for comprehensive student record management within a study group. It provides tools for managing student data including adding, editing, deleting, searching, and sorting students. All data is stored in a text file, allowing persistent storage and reloading when the application starts.

The application is operated via a simple console menu where the user can intuitively navigate between individual functions.

User features:

  • Display group – lists all students with numbering and full details
  • Edit group – add or remove a student
  • Search student – by surname, name, age, field of study or ID number, with partial match support
  • Sort students – by surname, name, age, field of study or ID number
  • File management – option to load an alternative source file
  • Exit program – saves current changes back to the file

Benefits and features:

  • Simple and clear console menu
  • Persistent data storage in a text file
  • Easy extensibility with new attributes
  • Modular design thanks to object-oriented architecture
  • Input validation minimizes user errors

What I learned from this project:

  • Basics of object-oriented programming in Java
  • Working with dynamic collections (ArrayList)
  • File reading and writing (I/O operations)
  • Text manipulation and searching
  • Building an interactive console menu
  • Basic algorithms (sorting, searching)
  • Input validation and exception handling
